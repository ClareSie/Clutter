# 方法  
## 種類隔離  
依服務提供與使用區分  

## 用戶隔離    
多租戶配合docker隔離  
1.每個租戶有獨立的服務和獨立的數據庫  
2.每個租戶有共用的服務和獨立的數據庫  
3.每個租戶有共用的服務和共用的數據庫  

## docker  
1.隔離    
應用間不互相影響  
2.可移植  
開發測試環境搭建便利一致  
3.輕量高效  
虛擬機龐大，應用級別隔離較不耗資源，部署也快
